Line Drawing Algorithms

Line drawing is a fundamental operation in computer graphics. A line drawing algorithm determines which pixels should be illuminated to best represent a line between two points on a discrete pixel grid (raster display). Since a continuous line cannot be perfectly represented on a pixel grid, line drawing algorithms use various techniques to approximate the line with the best possible visual result. In General two line drawing algorithms are used:-

  • Digital Differential Analyzer.
  • Bresenham's Algorithm.

Key Objectives:

  • Accuracy: The line should be as close as possible to the mathematical line.
  • Efficiency: The algorithm should be fast and use minimal computational resources.
  • Smooth Appearance: The rendered line should appear smooth and continuous to the human eye.
  • Consistency: Lines should be drawn consistently regardless of direction.

DDA (Digital Differential Analyzer) Algorithm

The DDA (Digital Differential Analyzer) algorithm is one of the simplest line drawing algorithms. It is based on the concept of calculating pixel positions using the differential equation of the line.

Principle:

The DDA algorithm uses the slope of the line to determine which pixels to illuminate. It moves from one pixel to the next along the line by incrementing either the x or y coordinate by 1, and calculating the corresponding coordinate on the other axis.

Algorithm Steps:

  1. Read the two endpoints of the line: (x₀, y₀) and (x₁, y₁)
  2. Calculate: dx = x₁ - x₀ and dy = y₁ - y₀
  3. Determine the number of steps: steps = max(|dx|, |dy|)
  4. Calculate increments:
    • x_increment = dx / steps
    • y_increment = dy / steps
  5. Initialize x = x₀ and y = y₀
  6. For i = 0 to steps:
    • Plot pixel at (round(x), round(y))
    • x = x + x_increment
    • y = y + y_increment
  7. Repeat the step 6 until x = x1
  8. When x = x1, end algorithm.

Advantages:

  • Simple to understand and implement
  • Works well for all slopes
  • No floating-point calculations required for plotting

Disadvantages:

  • Requires floating-point arithmetic
  • Rounding errors can accumulate
  • Slightly slower than Bresenham's algorithm

DDA Algorithm Code (Pseudocode):


void DDA(int x0, int y0, int x1, int y1)
{
float dx = x1 - x0;
float dy = y1 - y0;
int steps = (int)max(abs(dx), abs(dy));
float x_inc = (dx / steps);
float y_inc = (dy / steps);

float x = x0;
float y = y0;
for (int i = 0; i <= steps; i++)
{
plot_pixel(round(x), round(y));
x += x_inc; y += y_inc; }
}

DDA Example:

Draw a line from (0, 0) to (8, 4)

Calculations:

  • dx = 8 - 0 = 8
  • dy = 4 - 0 = 4
  • steps = max(8, 4) = 8
  • x_inc = 8/8 = 1
  • y_inc = 4/8 = 0.5

Bresenham's Line Drawing Algorithm

Bresenham's Algorithm is an efficient line drawing algorithm that uses only integer arithmetic. It was developed by Jack Bresenham and is widely used in computer graphics because of its speed and accuracy.

Principle:

Bresenham's algorithm determines which pixels should be selected to form a close approximation to a straight line between two endpoints. It uses the concept of error term to decide whether the next pixel should be plotted above or below the ideal line.

Algorithm Steps (for slope between 0 and 1):

  1. Read the two endpoints of the line: (x₀, y₀) and (x₁, y₁)
  2. Calculate: dx = x₁ - x₀ and dy = y₁ - y₀
  3. Calculate initial decision parameter: p = 2*dy - dx
  4. Initialize: x = x₀, y = y₀
  5. Plot pixel at (x, y)
  6. For i = 0 to dx:
    • If p < 0:
      • x = x + 1
      • p = p + 2*dy
    • Else:
      • x = x + 1
      • y = y + 1
      • p = p + 2*dy - 2*dx
    • Plot pixel at (x, y)

Advantages:

  • Uses only integer arithmetic (very fast)
  • More efficient than DDA algorithm
  • No rounding errors
  • Produces accurate and smooth lines
  • Widely used in hardware implementations

Disadvantages:

  • Slightly more complex than DDA
  • Requires handling of different slope cases

Bresenham's Algorithm Code (Pseudocode):


void Bresenham(int x0, int y0, int x1, int y1)
{
int dx = abs(x1 - x0);
int dy = abs(y1 - y0);
int x = x0, y = y0;
int x_inc = (x0 < x1) ? 1 : -1;
int y_inc = (y0 < y1) ? 1 : -1;

if (dx > dy)
{
int p = 2*dy - dx;

for (int i = 0; i < dx; i++)
{ plot_pixel(x, y);
x += x_inc;

if (p < 0)
{ p += 2*dy; }
else
{ y += y_inc; p += 2*dy - 2*dx; }
}
} else { int p = 2*dx - dy;
for (int i = 0; i < dy; i++)
{
plot_pixel(x, y);
y += y_inc;
if (p < 0)
{ p += 2*dx; }
else
{ x += x_inc; p += 2*dx - 2*dy; } } } }
